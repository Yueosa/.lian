## bin 目录说明

这个目录对应的是你系统里的 `$HOME/.local/bin`（我在 Hyprland 配置里把 `$script_dir` 指向这里）。

我的实际推荐是：

- `~/.local/bin` 只放软链（不要把“真实脚本”直接丢进去）
- 快捷键/启动项统一调用 `~/.local/bin/...`（路径稳定、好维护）
- “真实脚本/配置源文件”放在 `~/.config/...`（或本仓库 `~/.lian/...` 再软链到 `~/.config`）

那为什么还会有这个 `bin/`？

这里的文件是一些“可选的 wrapper/跳板脚本”，用途是：

- 你不想记住很深的路径（例如 `~/.config/waybar/scripts/...`）
- 或者你在抄作业时还没把目录软链到 `~/.config`，但又想先把快捷键跑起来

它们会把调用转发到本仓库里对应的真实脚本。

默认情况下，我仍然更推荐你直接软链 `~/.config` 下的真实脚本到 `~/.local/bin`。

如果你跟我一样把 `~/.config/<软件>` 也软链到了 `~/.lian/<软件>`，那么让 `~/.local/bin` 直接指向 `~/.lian/...` 会更清晰：`~/.lian` 才是唯一的源，`~/.config` 只是分发出来给应用读取。

另外：如果你遇到 `permission denied`，记得给脚本补执行权限（例如 `chmod +x bin/*/*`）。
